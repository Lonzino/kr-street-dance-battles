**조사 일자**: 2026-04-23
**범위**: 한국 스트릿 댄스 씬의 대회·크루·커뮤니티 관련 서비스

## 핵심 발견

1. **직접 경쟁사 1곳 존재**: [DanceCode (dancecode.kr)](https://www.dancecode.kr/) — WillUEntertainment 운영, 앱까지 있음
2. **우리 데이터 확인**: 11배틀 중 3건이 이미 DanceCode를 등록 창구로 사용 중
   - `aim-high-2026-street-open-style`
   - `cidc-2026-gyeonggi-south`
   - `jecheon-youth-dance-festival-2026`
3. **실질 경쟁은 플랫폼이 아닌 관행**: 인스타 DM + 구글폼 + 카톡 단톡 조합이 메인 대체재
4. **해외 플랫폼**은 **학원 댄스(재즈·발레)** 중심 — **스트릿 배틀 시장에는 부재**

## DanceCode 상세 분석 (직접 경쟁)

### 확인된 기능
- 행사 등록 (주최자)
- 참가/관람 신청
- **실시간 중계** (우리는 없음)
- 행사 영상 아카이브 (YouTube 연동)
- 댄서 활동 기록 / 후원·광고 연결 (공식 소개 문구)
- **MakeCode** — 창작 댄스 프로젝트 발굴 프로그램
- 모바일 앱 (Google Play)

### 알 수 없는 (더 조사 필요)
- 실제 MAU / 활성 주최자 수
- 결제 통합 여부
- 현장 체크인 기능
- 가격 모델 (무료 주장, 실제는 수수료 가능)

### 우리 데이터의 시사점
우리 데이터 11건 중 3건 등록 창구로 사용됨 → **시장 점유율 30% 정도 도달한 플레이어**로 추정. 무시할 수 없음.

### 우리 대비 DanceCode 강점
- ✅ **선점자 유리** (이미 운영 중, 주최자 다수 사용)
- ✅ 앱 (우리는 웹만)
- ✅ 실시간 중계 (우리는 없음)
- ✅ 후원·광고 시스템 (비즈니스 모델 있음)
- ✅ 한국 공식 주최 행사에서 채택 (제천 청소년 페스티벌 등 공공)

### 우리 대비 DanceCode 약점 (추정)
- ❓ **K-pop·학원 댄스 포함 범용** → 스트릿 씬 깊이 부족 가능성
- ❓ **상업적 운영사** (WillU Entertainment) → 씬의 감정적 귀속 약할 수 있음
- ❓ **오픈소스 X** → 커뮤니티 기여 불가
- ❓ 디자인·UX 경쟁력 (직접 확인 필요)
- ❓ 비보잉 전문성 (주최자 퓨전엠씨·갬블러즈 같은 씬 네이티브 관계 여부)

## 우리 차별화 포인트 (근거 기반)

### 구조적 차별
1. **오픈소스 (GitHub)** — 씬 사람 누구나 기여 가능, 투명성
2. **1인 운영 + 커뮤니티** — 상업 사업자와 다른 스탠스
3. **스트릿 배틀 전용** — K-pop·학원 댄스 섞임 X
4. **무료 + 후원 모델** — 수수료 부담 0 (DanceCode도 무료 주장하지만 실제 미확인)
5. **깃에 데이터 공개 (CC0)** — 연구자·개발자 재사용 가능

### 현재 코드 상 우위
- ✅ LLM 자동 정보 추출 (Claude API) — DanceCode에 없음
- ✅ 현장 QR 체크인 — 확인되지 않음
- ✅ 크루↔댄서 자동 연결 (fuzzy match + alias)
- ✅ JSON-LD + og:image 등 SEO 준비
- ✅ 개인정보보호법 준수 (회원 탈퇴·익명화 구현)

### 현재 코드 상 열위
- ❌ 앱 없음 (웹만)
- ❌ 실시간 중계 없음
- ❌ 유저 없음 (DanceCode는 선점자 유리)
- ❌ 결제 내재화 없음 (의도적이지만 주최자 입장에선 불편)

## 포지셔닝 전략

### 가능한 포지션 3가지

#### A. **"스트릿 씬 전용 오픈 아카이브"** (추천)
- DanceCode가 범용 → 우리는 **스트릿 깊이**
- 오픈소스·CC0 데이터 → 씬이 소유한 플랫폼
- 모토 예: "스트릿 댄서들이 만들고 운영하는 아카이브"
- 타겟: 비보이·팝퍼·락커·왁커·보거·크럼퍼 (K-pop·학원 X)

#### B. **"대회 운영 무료 도구"** (기능 경쟁)
- DanceCode와 정면 경쟁
- 참가 신청·체크인·결과 다 내재화
- 차별: 오픈소스 + 무료
- 위험: DanceCode 이미 존재, 후발 주자 불리

#### C. **"댄서 개인 포트폴리오"** (LinkedIn-ish)
- 참가 기록·결과가 자동 축적되는 댄서 CV
- 기존 경쟁사 공백 영역
- 단점: 사용자 없으면 가치 0 (Cold Start 심각)

**추천**: **A + C 조합** — "스트릿 씬 전용 아카이브 + 댄서 개인 포트폴리오 자동 축적".

DanceCode와 정면 B 경쟁 X — 선점자 유리 + 우리 리소스 부족.

## Atomic Network 전략과 경쟁 관계

### 광주 LINE UP 씬 진입 시

**빛고을댄서스 입장에서 비교할 것:**

| 질문 | DanceCode | 우리 | 구글폼+인스타 (현재) |
|------|-----------|------|---------------------|
| 신청 받기 | 있음 | 있음 | 수동 |
| 참가비 받기 | 아마 있음 | 외부 | 외부 |
| 명단 정리 | 있음 | 자동 | 엑셀 수동 |
| 체크인 | 불확실 | QR | 종이 |
| 크루 페이지 | 범용 | 스트릿 특화 | 없음 |
| 시즌 기록 아카이브 | 있음 | 자동 | 인스타 자체 |
| **가격** | 추정: 수수료·광고 | **무료** | 무료 |
| **코드 공개** | X | **공개** | N/A |

**차별 메시지**:
> "DanceCode는 범용 댄스 플랫폼이지만, 우리는 스트릿 배틀 전용이라 필요한 부문(1on1 비보잉, 크루배틀 등)을 정확히 반영합니다. 오픈소스라 광주 씬이 원하는 기능을 요청하면 바로 반영 가능해요."

### Pivot 시나리오 추가

만약 **DanceCode가 빛고을댄서스 이미 사용 중**이면:
- 굳이 교체 요구 X
- 대신 **우리는 아카이브·검색·댄서 프로필에 집중**
- 양쪽 공존 가능 (multi-homing — HBR 2019)
- 결과만 우리에 자동 기록되게 API 통합 제안

## 결론

### 좋은 소식
- ✅ 스트릿 배틀 **전용** 플랫폼은 명확히 부재 (DanceCode도 범용)
- ✅ 해외 경쟁자 없음 (지역·언어 장벽)
- ✅ 시장 존재 증명 (DanceCode 운영 중)

### 나쁜 소식
- ❌ 선점자 DanceCode 존재 (우리 데이터 27%가 사용)
- ❌ 후발 주자로서 차별화 필수
- ❌ 주최자 대부분이 아직 **구글폼 + 인스타** 관성 (플랫폼 교체 저항)

### 전략 확정
1. **DanceCode와 정면 경쟁 X** (리소스 낭비)
2. **스트릿 전용 + 오픈 아카이브**로 포지셔닝
3. **atomic network (광주 LINE UP) 먼저** — 한 씬에서 깊이 구축
4. **공존 가능성 열어둠** (multi-homing)
5. **월 1회 경쟁사 모니터링**
