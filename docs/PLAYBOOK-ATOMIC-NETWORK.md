**시나리오 A — 답장 옴 (긍정)**:
- [ ] 30분 통화 또는 광주 직접 만남 약속

- [ ] 5분 영상 데모 만들어서 보냄 (Loom·QuickTime)

**시나리오 C — 무답 7일**:
- [ ] **다음 atomic network 후보로 전환**: B (서울 비보잉 — Jinjo Crew 또는 R16 KOREA), C (부천 힙합 페스티벌 — 시간 임박해서 단발 검증용)
- [ ] 동시에 LINE UP 시즌 11 참가자(댄서)에게 직접 DM 가능 (운영자 패스)

#### Day 5-7: 첫 만남 (시나리오 A 진행 시)
- [ ] 만남 — 어젠다 따라 진행 (#템플릿 2)
- [ ] **녹음 동의 받기** (분석용, 외부 공개 X)
- [ ] 미팅 후 24시간 안에 약속 사항 정리해서 보냄
- [ ] 첫 합의 1건 도출:
  - 예: "시즌 11 신청을 사이트로 받기로"
  - 예: "참가자 명단 export만 사이트 사용"
  - 예: "체크인만 사이트 사용"
  - 작은 합의도 OK — 첫 사용 시작이 핵심

- [ ] **가장 중요**: 운영자가 "사이트가 더 편하다"고 느끼는 순간이 언제인가?

#### Day 11-13: 피드백 → 코드 수정
- [ ] 운영자 피드백 1순위로 처리 (가설 X, 그들이 막힌 것만)

#### Day 14: 검증 회고
- [ ] 운영자 인터뷰 (#템플릿 3)
- [ ] 다음 시즌도 사이트 쓸지 약속 받기
- [ ] 본인 회고: 무엇이 작동, 무엇이 안 작동
- [ ] 다음 atomic network로 갈 준비 (또는 같은 atomic 더 깊이)

### 상황별 플레이북

#### "사이트보다 인스타 DM/구글폼이 더 편함"
- 진짜 그런가 검증: 운영자 옆에서 인스타 DM 신청 받는 시간 측정
- 사이트가 진짜 빠르다면: 데이터로 보여줌
- 사이트가 진짜 안 빠르다면: 그 부분 즉시 고치기

#### "참가자가 회원가입을 귀찮아함"
- Kakao/Google 로그인 외 더 빠른 옵션 검토 (전화번호 OTP — 추후 작업)
- 또는 비회원 신청 허용 (admin이 사후 매칭)

#### "결제는 어떻게 받지?"
- 외부 (계좌·토스 송금) 안내 명확히
- 입금 확인 admin 워크플로 데모

#### "참가자가 사이트 못 찾아옴"
- 빛고을댄서스 인스타 게시물에 사이트 URL 명시 부탁
- 단축 URL (bit.ly) 제공
- QR 코드 포스터에 인쇄 제안

## Pivot 신호 (포기 X, 방향 전환)

다음 중 3개 이상 동시 발생 시 atomic network 재선택:

- 운영자 7일 무답
- 미팅 가졌는데 첫 합의 0건
- 합의 받았는데 1주 안에 사용 0건
- 사이트가 외부 폼보다 명백히 불편한 게 발견됨
- 본인이 광주 갈 시간·체력 부족

**Pivot 우선순위**:
1. **다른 광주 주최자** (CIDC 광주·강원 청소년 페스티벌)
2. **서울 비보잉** (Jinjo Crew → R16 또는 BBIC 운영진)
3. **부천 힙합 페스티벌 5/4** (단발 — 검증보다 데이터 수집용)

대회 운영 도구를 무료로 제공하고 있는데, 시즌 11에 베타 사용자로 한 번 같이 운영해보시면 어떨까 해서 연락드렸습니다.

지원 가능한 것:
• 사이트에서 직접 참가 신청 받기 (구글폼 대체)

• 참가비 입금 확인 한 화면에서 관리
• 현장 QR 체크인

부담 0, 결제 X, 운영자 시간 줄이는 게 목적이에요.
30분 통화 또는 광주 한 번 가능할까요?

luveach@gmail.com

**왜 이 톤?**
- 자기 소개 짧게 (영업톤 X)
- 이미 그들 위해 뭐 했는지 (크루 페이지) — 호의의 증거
- 도움될 수 있는 것 명시 (모호한 협업 X, 구체적 도구)
- 부담 명시 ("부담 0, 결제 X")
- 약속 작게 (30분 통화)

### 템플릿 2: 첫 미팅 어젠다 (30~60분)

```
1. (5분) 자기 소개 + 사이트 만든 이유
   "댄스 정보가 인스타에 흩어져 있어서 모으고 싶었다"

2. (15분) 그쪽 이야기 듣기 — 가장 중요
   - 시즌 11 운영 어떻게 하시나요?
   - 가장 시간 많이 드는 부분?
   - 가장 짜증나는 부분?
   - 시즌 10에서 뭐가 잘 안 됐어요?
   ⚠️ 본인 사이트 자랑 X. 듣기만.

3. (10분) 사이트 짧은 데모
   - 신청 페이지 → 명단 자동 → 체크인까지 1분
   - "이게 그쪽 운영에 도움될 것 같으면 같이 써보시고, 아니면 안 써도 됩니다"

4. (10분) 첫 합의 도출
   - 가장 작은 시도 제안
   - 예: "시즌 11 사이트로 신청 받아보시겠어요?"
   - 부담 명시: "안 맞으면 시즌 12부터 안 쓰셔도 됩니다"
   - **합의 안 되면 Pivot 검토**

5. (5분) 다음 단계 약속
   - 카톡 단톡방 만들기 (운영자 + 본인 + 필요시 다른 멤버)
   - 다음 통화/만남 예정일 확정
   - 24시간 안에 본인이 보낼 자료 명시
```

### 템플릿 3: Day 14 회고 인터뷰 (45분)

```
[먼저 감사 인사 + 시즌 11 운영 어땠는지 기념]

1. 사이트 처음 들어왔을 때 인상
   "회원가입까지 몇 초 걸렸어요?"
   "헷갈리는 부분 있었어요?"

2. 신청 받기 — 외부 폼 대비
   "구글폼이랑 비교해서 더 빠른가요? 더 느린가요?"
   "참가자들 반응?"

3. 결제·체크인
   "현장에서 QR 스캔 해보셨어요? 어땠어요?"
   "입금 확인 워크플로 잘 됐어요?"

4. 가장 중요한 질문 — Sean Ellis Test
   "만약 이 사이트가 갑자기 사라지면 어떨 것 같아요?"
   - "엄청 아쉬울 것" → PMF 신호
   - "별로 안 아쉬움" → 가치 없음, 다음 atomic으로

5. 다음 시즌 (시즌 12)도 사용 의향?
   - "네" → Phase B-2 진입
   - "고민" → 무엇이 부족? 1주 안에 고치고 다시 제안
   - "아니요" → 인터뷰로 진짜 이유 파악, 다른 atomic으로

6. 추천
   "이 도구를 다른 주최자한테 추천하실 거 같아요?"
   - 광주 다른 주최자 소개 받기 (CIDC 등)
   - 서울 친한 운영자 있나?
```

**Sean Ellis Test 기준**: "엄청 아쉬울 것"이 응답자의 **40% 이상**이면 PMF 신호. (1차 출처: Sean Ellis가 Dropbox·LogMeIn에서 사용)
